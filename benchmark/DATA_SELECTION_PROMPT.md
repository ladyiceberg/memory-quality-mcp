# 数据筛选 Prompt · 从真实对话中提取 Benchmark 样本

> 用途：让模型从大量真实 Chatbot 对话中，筛选出适合用于「AI 记忆质量 Benchmark」的对话片段
> 使用方式：把这个 Prompt 发给模型，同时附上你的对话数据

---

## Prompt 正文

---

你是一个数据标注专家，正在为「AI 记忆质量评估」项目筛选训练数据。

**背景说明**

AI 助手在对话结束后会自动把值得记住的内容写入持久记忆。但并不是所有对话都值得提取记忆，有些对话包含应该被记住的重要偏好，有些包含不应该被记住的临时状态，有些则会导致 AI 产生错误的记忆。

你的任务是从我提供的对话数据里，**筛选出能代表不同记忆质量场景的对话片段**，供后续评估 AI 记忆提取质量使用。

---

**需要筛选的 6 种场景类型**

请从数据中各找 5-8 条（如果数量不足则尽量多找）：

**类型 A：明确值得记住的偏好/背景**
- 用户明确说了自己的习惯、偏好、身份背景、工作方式
- 例：「我是做数据分析的，平时用 Python」「我不喜欢太长的回答」「我有乳糖不耐受」
- 这类对话提取出的记忆应该是高质量的，应该被保留

**类型 B：临时状态（不应被记住）**
- 用户提到的是当下的一次性状态，而非长期事实
- 例：「我今天有点头疼，状态不好」「这周在赶一个 deadline」「刚吃完饭有点困」
- 这类对话不应该被提取为持久记忆，如果 AI 记了就是错误

**类型 C：用户纠正 AI 的行为**
- 用户明确表达对 AI 做法的不满，或指出 AI 做错了什么
- 例：「你每次都这样，我说了不要加总结」「别帮我做决定，给我选项就行」「上次你说的是错的」
- 这类对话应该被记忆，让 AI 下次不再犯同样的错

**类型 D：项目/任务背景（时效性强）**
- 用户提到正在进行的项目、任务、计划，包含明确的时间信息
- 例：「我在做一个电商小程序，下个月要上线」「这个季度我们在做品牌升级」
- 这类记忆有价值，但会随时间过时，适合测试时效性评分

**类型 E：AI 可能过度解读的对话**
- 用户只是随口一说，但 AI 可能会把它当成长期偏好记下来
- 例：用户说「今天想早点睡」→ AI 可能错误记成「用户习惯早睡」
- 例：用户说「最近在学 React」→ AI 可能错误记成「用户是前端开发者」
- 这类场景用于测试 AI 是否存在「过度解读」问题

**类型 F：前后矛盾的对话对**
- 同一用户在不同对话里说了相反或矛盾的事情
- 例：一次说「我喜欢简洁的风格」，另一次说「可以写详细一点吗」
- 例：一次说「我是初学者」，另一次展现出很强的专业知识
- 注意：需要成对提供（对话 A + 对话 B），标注它们哪里矛盾

---

**筛选标准**

每段对话应满足以下条件：
1. **长度**：4-12 轮（用户+模型各算一轮），太短信息不足，太长请截取关键部分
2. **独立性**：每段对话能独立理解，不需要其他上下文
3. **包含关键信息**：对话里有能被提取为记忆的信息（不管应不应该被记）
4. **排除**：纯粹的信息查询（「帮我查一下」「翻译这段话」），这类对话没有记忆价值

---

**输出格式**

对每条筛选出的对话，输出以下结构：

```json
{
  "scenario_type": "A/B/C/D/E/F",
  "scenario_label": "中文标签（如：明确偏好、临时状态、AI纠错等）",
  "expected_action": "should_remember / should_not_remember / conflicting",
  "key_signal": "这段对话里最关键的一句话或信息点（你认为 AI 会提取什么）",
  "potential_issue": "如果 AI 提取了记忆，可能出现什么问题（仅类型B/E/F需填写）",
  "messages": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ]
}
```

类型 F（矛盾对）需要额外字段：
```json
{
  "scenario_type": "F",
  "conflict_description": "两段对话在哪里矛盾",
  "dialogue_a": { "messages": [...] },
  "dialogue_b": { "messages": [...] }
}
```

---

**数量目标**

| 类型 | 目标数量 |
|------|---------|
| A：明确偏好/背景 | 5-8 条 |
| B：临时状态 | 5-8 条 |
| C：AI 纠错 | 3-5 条 |
| D：项目背景 | 3-5 条 |
| E：过度解读风险 | 3-5 条 |
| F：前后矛盾对 | 2-3 对 |
| **合计** | **约 25-40 条** |

---

**以下是我的对话数据，请开始筛选：**

[在此处粘贴你的对话数据]

## 使用说明

1. 把上面的 Prompt 正文完整复制
2. 替换最后一行 `[在此处粘贴你的对话数据]` 为你的真实数据
3. 发给模型（建议用 Claude 或 GPT-4o，上下文窗口大的）
4. 模型返回筛选结果后，把 JSON 保存为文件发给我
5. 我来运行提取脚本，生成记忆文件和 Benchmark 条目

## 注意事项

- 如果你的数据量很大（>100 条对话），建议分批发送，每批 30-50 条
- 筛选结果不需要完美，有遗漏没关系，我们会人工复核
- 类型 F（矛盾对）最难找，如果找不到够的数量，类型 A/B/E 可以多找几条补充
